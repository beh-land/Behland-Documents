# Contributing to BehLand Documentation

We welcome contributions to the BehLand documentation! Whether you want to help improve existing content or add new materials, your contribution is valuable.

## How to contribute
1. Fork the repository and create your branch from `main`.
2. Make your changes or additions to the documentation.
3. Commit your changes and push your branch to your forked repository.
4. Open a pull request (PR) to the main repository.

## Reporting Issues
If you find any issues or bugs in the documentation, please report them by opening an issue in the repository.

- For bugs or corrections, use the **Bug/Doc Issue** template.
- For suggestions, use the **Suggestion** template.
- For security-related reports, use the **Security Report** template and contact us via our official email or Telegram.

## Code of Conduct
Please follow the project's code of conduct and contribute respectfully to the community.
